Gestão Escolar · Next.js + Prisma

Documentação Rápida

Painel administrativo com autenticação JWT (cookie httpOnly), RBAC via permissões/grupos e CRUDs completos.

Pré-requisitos

  • Node 18+ e npm
  • PostgreSQL acessível (ex.: Neon)
  • Variáveis em .env.local

Variáveis de Ambiente

Crie .env.local a partir de .env.example:

  • DATABASE_URL: string de conexão (pooler Neon)
  • AUTH_SECRET: chave longa para assinar JWT
  • SEED_MASTER_EMAIL, SEED_MASTER_PASSWORD, SEED_MASTER_NAME
  • DIRECT_URL (opcional, sem pooler)

Instalação / Dev

1
npm install
2
Reset (opcional):
npx prisma db execute --stdin <<'SQL' DO $$ DECLARE r RECORD; BEGIN FOR r IN (SELECT tablename FROM pg_tables WHERE schemaname = 'public') LOOP EXECUTE 'DROP TABLE IF EXISTS public.' || quote_ident(r.tablename) || ' CASCADE'; END LOOP; END $$; SQL
3
Aplicar schema: npx prisma db execute --file prisma/sql/init.sql
4
Gerar client: npx prisma generate
5
Seed (grupos/permissões/master): node prisma/seed.ts
6
Dev server: npm run dev

Login padrão (seed): master e senha definidos em SEED_MASTER_*.

Build / Produção

  • Aplicar SQL canônico: prisma/sql/init.sql
  • npx prisma generate
  • Rodar seed se precisar criar permissões/grupos/master
  • npm run build e npm run start
  • Garantir escrita em public/uploads ou configurar storage nas rotas de upload

Rotas Principais

Autenticação

/login · sessão JWT em cookie httpOnly

Dashboard

/dashboard

Cadastros

Alunos, Responsáveis, Turmas, Disciplinas, Matrículas

Acadêmico

Turma x Disciplina, Prof. x Componente, Períodos, Frequência, Avaliações, Notas, Fechamento

Segurança

Usuários, Grupos, Permissões (RBAC aplicado em UI e servidor)

Configurações

Escola (logo/nome/contatos) e popular/resetar base (demo)

RBAC

  • Permissões em permissions, grupos em user_groups, vínculos em group_permissions
  • Master: todas; Admin: todas exceto Configurações; Secretaria: cadastros principais; Professor: leituras + frequência/avaliações
  • Enforcement: server actions/rotas e visibilidade de menu/botões

Uploads

  • Avatar: /api/upload/avatar (padrão em public/uploads)
  • Logo: /api/upload/logo
  • Tipos: jpeg/png/webp/gif, até 5MB

Saúde

/api/health executa SELECT 1 via Prisma.

SQL canônico: prisma/sql/init.sql · Prisma Client gerado sem migrations.
